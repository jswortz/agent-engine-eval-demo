The OpenTelemetry instrumentation architecture for Vertex AI Agent Engine provides automatic, zero-code observability through a hierarchical span structure that captures every phase of agent execution.

Instrumentation is activated by setting four environment variables in the agent's set_up method. The primary toggle GOOGLE_CLOUD_AGENT_ENGINE_ENABLE_TELEMETRY set to true activates the built-in tracing. OTEL_SERVICE_NAME assigns a logical service identifier, in this case demo-finance-agent. OTEL_PYTHON_LOGGING_AUTO_INSTRUMENTATION_ENABLED enables automatic log correlation with trace context. OTEL_INSTRUMENTATION_GENAI_CAPTURE_MESSAGE_CONTENT enables capturing the actual prompt and response content within spans.

When a query arrives, Agent Engine creates a root span representing the entire agent invocation. This root span contains a child span for the ReAct reasoning loop, which orchestrates the agent's think-act-observe cycle. Within the reasoning loop span, two types of child spans are created dynamically. LLM generation spans capture each call to the Gemini 2.5 Flash model, recording input token count, output token count, and generation latency. Tool execution spans capture each invocation of registered tools like get_billing_status, recording the tool name, input arguments, output result, and execution duration.

The reasoning loop may iterate multiple times if the agent needs to call tools and then reason about the results, creating a nested tree of alternating LLM and tool spans under the loop parent. Each iteration adds another layer to the span hierarchy, producing a waterfall visualization that reveals the exact sequence and timing of agent decisions.

All spans are automatically exported to Google Cloud Trace without any custom exporter configuration. In the Cloud Trace console, developers can view the full span waterfall to identify bottlenecks, distinguishing between slow LLM inference, slow tool calls, and orchestration overhead. Key metrics available out of the box include step-by-step latency breakdowns, total token consumption per reasoning step, and invocation success versus failure counts.
