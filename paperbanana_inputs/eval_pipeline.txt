The evaluation pipeline receives agent session logs containing prompt-response pairs extracted from the deployed FinanceAgent and processes them through two parallel evaluation branches that converge into a unified quality assessment.

The input to the pipeline is a pandas DataFrame with three columns: the original user prompt, the agent's generated response, and a human-written reference answer. This DataFrame flows into the Vertex AI EvalTask, which orchestrates the evaluation process.

The first branch applies a custom Model-as-a-Judge rubric using the PointwiseMetricPromptTemplate. This rubric defines two qualitative criteria. The helpfulness criterion requires that the response directly and accurately answer the user's initial request. The conciseness criterion requires that the response be brief, with no rambling or unnecessary pleasantries. For each prompt-response pair, an evaluator LLM reads the content and assigns a rating on a 1 to 5 scale: a score of 1 means the response fails to answer the request entirely, a score of 3 means the response is passable but contains irrelevant content or requires cognitive effort, and a score of 5 means the response is perfect with an accurate and concise answer. The evaluator also produces a written explanation justifying its score. The two criteria ratings are combined into a single agent_quality_score.

The second branch runs an exact_match base evaluation that performs algorithmic string comparison between the agent's response and the reference answer, producing a binary 0 or 1 result. This serves as a deterministic baseline complementing the subjective rubric scoring.

Both evaluation branches feed their results back into the EvalTask, which assembles a unified metrics_table DataFrame. Each row contains the original prompt, the agent response, the numeric quality score from the custom rubric, the evaluator's written explanation, and the exact match result. This combined output captures both the nuanced qualitative judgment and the objective baseline comparison in a single structured record.
