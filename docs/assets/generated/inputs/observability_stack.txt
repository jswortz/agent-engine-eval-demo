This diagram shows the complete observability stack for an ADK agent deployed on Agent Runtime.

Layer 1 - Agent Execution (bottom):
- ADK Agent (finance_agent) running on Agent Runtime
- AdkApp wrapper with enable_tracing=True
- Environment variables: OTEL_SERVICE_NAME, OTEL_SEMCONV_STABILITY_OPT_IN, OTEL_INSTRUMENTATION_GENAI_CAPTURE_MESSAGE_CONTENT
- Auto-instrumented spans: invocation → invoke_agent → call_llm → generate_content + execute_tool

Layer 2 - OpenTelemetry Pipeline (middle):
- OTEL SDK (v1.38.0+) with gen_ai.* semantic conventions
- Three exporters fan out from the OTEL pipeline:
  1. OTLP exporter → Cloud Trace (spans with resource attributes: cloud.platform, service.name, gen_ai.agent.name)
  2. Logging exporter → Cloud Logging (structured logs with trace context correlation)
  3. Monitoring exporter → Cloud Monitoring (metrics: model calls, tool calls, token usage, latency)

Layer 3 - Observability Services (top):
- Cloud Trace: Trace Explorer with DAG graphs, waterfall views, span detail
- Cloud Logging: Logs Explorer with structured queries, log-trace correlation
- Cloud Monitoring: Dashboards with agent-specific metrics (Models, Tools, Usage tabs)
- Agent Console: unified dashboard combining all three services into Overview, Models, Tools, Usage, Logs, Traces, Evaluation tabs

The diagram emphasizes that zero custom instrumentation code is needed — all spans, logs,
and metrics are auto-generated by the AdkApp wrapper and Agent Runtime platform.
